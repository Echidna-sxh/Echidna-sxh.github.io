librdkafkacpp 是一个 kafka 的 c++ 库，使用对象的创建使用工厂模式，自定义的逻辑处理一般使用回调函数。

libkafkacpp 提供的接口基本覆盖了 kafka 原生提供的功能，但也比较琐碎，可以进行进一步封装。下面是对上一章的示例的详细解读。

# 生产者

  • 首先原生生产者是 RdKafka::Producer, 其需要的其他变量有：
  1. 目标 broker (s)
  2. 话题对象
  3. 回调函数 (可选)
  4. 写入的目标 partition
  • 其初始化流程为：

std::string errstr="";
//创建全局配置对象指针
RdKafka::Conf *conf = RdKafka::Conf::create(RdKafka::Conf::CONF_GLOBAL);
//创建话题配置对象指针
RdKafka::Conf *tconf = RdKafka::Conf::create(RdKafka::Conf::CONF_TOPIC);
  • 这里应用工厂模式，创建了两个配置对象。rdkafka 的类大多使用大量纯虚函数，然后使用静态函数创建对象。
if (conf->set("metadata.broker.list", brokers_, errstr) != RdKafka::Conf::CONF_OK){//
        std::cerr << "RdKafka conf set brokerlist failed :" << errstr.c_str() << std::endl;
    }
  • 关于 conf->set 可以做的事较多，这里是设置 brokers，具体可以看附录
conf->set("dr_cb", &delivery_callback, errstr);//设置递送完包的回调
conf->set("event_cb", &event_callback, errstr);//设置事件回调
  • 可选，注意设置了 delivery_callback 时，一定要在发送消息后使用 poll () 来清空消息报告队列，因为 poll () 会调用上述两个回调函数。
kafka_producer_=RdKafka::Producer::create(conf,errstr);
  • 使用前面进行的配置创建生产者对象
topic_=RdKafka::Topic::create(kafka_producer_,topics_,tconf,errstr);
  • 使用生产者对象和话题以及话题配置创建话题对象

至此生产者及其话题可以用于发送消息，接下来的内容可以封装为一个单次发送信息函数，方便多次调用。

RdKafka::ErrorCode resp = kafka_producer_->produce(topic_, partition_,
                                                    RdKafka::Producer::RK_MSG_COPY /* Copy payload */,
                                                    const_cast<char *>(msg.c_str()), 
                                                    msg.size(),
                                                     NULL, NULL);

  • 第一个参数是前面的话题对象

  • 第二个参数是指定写入的 partition，可以设置 RdKafka::Topic::PARTITION_UA 来自动负载均衡，可以开个类继承 PartitionerCb 类重载 partitioner_cb 函数来自定义负载均衡策略。

  • 第三个参数指示生产选项，这里是拷贝负载模式 RK_MSG_COPY，还有用完即释放负载模式 RK_MSG_FREE，这里两个都是异步非阻塞的，还有 RK_MSG_BLOCK 模式，这会在消息队列满时阻塞

  • 第四个参数是要发送的信息，一般使用 char * 类型

  • 第五个参数是数据长度

  • 第六个参数是数据的 key，可以是空

  • 第七个参数是隐藏信息，用作上下文。

    函数原型如下：

virtual ErrorCode produce (Topic *topic, int32_t partition, int msgflags, void *payload, size_t len, const std::string *key, void *msg_opaque)=0

if (resp != RdKafka::ERR_NO_ERROR)
        std::cerr << "Produce failed: " << RdKafka::err2str(resp) << std::endl;
  • RdKafka::err2str (RdKafka::ErrorCode resp) 是将错误代码转化为可读错误。
while (run && kafka_producer_->outq_len() > 0) {
        std::cerr << "Waiting for " << kafka_producer_->outq_len() << std::endl;
        kafka_producer_->poll(1000);//由于是异步调用，是否成功是看回调函数的，这里poll会调用dr_cb函数
    }
  • 本次调用退出前处理完消息队列中的消息。

# 消费者

对于消费者，有一个消费者组的概念，同一消费者组的消费者无法重复消费同一 partition，librdkafkacpp 提供两种消费者的 api，分别为高级 api 和低级 api，高级 api 拥有更好的负载均衡策略，即会更好的分配给每个消费者 partition。

本示例使用低级 api，即指定 partition 进行消费。

RdKafka::Conf *conf=RdKafka::Conf::create(RdKafka::Conf::CONF_GLOBAL);
  • 同样是 GLOBAL 类型配置对象。
if(conf->set("bootstrap.servers",brokers_,errstr)!=RdKafka::Conf::CONF_OK){
        fprintf(stderr, "RdKafka conf set brokerlist failed : %s\n", errstr.c_str());
    }
  • 设置 brokers
if (conf->set("group.id", groupid_, errstr) != RdKafka::Conf::CONF_OK){
        fprintf(stderr, "RdKafka conf set group.id failed : %s\n", errstr.c_str());
    }
  • 设置消费者组 id
std::string strfetch_num="1024000";
    if(conf->set("max.partition.fetch.bytes", strfetch_num, errstr) != RdKafka::Conf::CONF_OK){
        fprintf(stderr, "RdKafka conf set max.partition failed : %s\n", errstr.c_str());
    }
  • 设置每次获取的最大数据长度
kafka_consumer_ = RdKafka::Consumer::create(conf, errstr);
  • 根据完成的配置创建消费者对象
  • 创建话题的配置对象
  • 设置每次连接自动重置到最小的 offset（即最早的信息）, 这个最小的 offset 不一定是 0, 因为 msg 可能会被过期删除掉
topic_ = RdKafka::Topic::create(kafka_consumer_, topics_, tconf, errstr);
  • 根据话题的配置对象创建话题对象
RdKafka::ErrorCode resp = kafka_consumer_->start(topic_, partition_, offset_);
  • 开启消费通道
RdKafka::Message *msg = NULL;//声明数据指针
    while(run_){
        msg = kafka_consumer_->consume(topic_,partition_,timeout_ms);//从目标partition获取对应话题信息
        handle_message(msg,NULL);//处理获取到的信息
        kafka_consumer_->poll(0);//在获取到下一条数据之前无限等待
        delete msg;
    }//如果handle时run没有置0则一直循环
  • 开始循环监听消息
kafka_consumer_->stop(topic_,partition_);
  • 和 start 对应，停用消费通道

之后销毁创建的两个对象，注意销毁后调用 RdKafka::wait_destroyed (5000); 来确保销毁所需时间。

低级消费者的函数原型如下：

virtual Message* RdKafka::Consumer::consume	(Topic * topic,int32_t 	partition,int timeout_ms )
  • 第一个参数是目标话题，第二个参数是目标 partition，第三个参数是等待接受消息的最长时间
